# Кейс №3: Анализ транзакционных аномалий и сравнительный аудит финтех-потока

## Блок: Финтех / Управление рисками
**Заказчик:** Руководитель отдела финтеха.

### Бизнес-задача (ТЗ):
> *"Необходимо сформировать детализированный реестр транзакций за отчетный период (май), отражающий реальный объем операций без искажений. Для проведения сравнительного аудита и выявления аномально крупных транзакций требуется вывести показатели каждой отдельной операции, а в соседней колонке рассчитать среднее историческое значение объема по всей выборке за этот период. Детализированные строки отчета не должны быть агрегированы или схлопнуты."*

---

## Исходная структура СУБД (Таблица `raw_transactions`):
*   `tx_id` (INT) — Первичный ключ транзакции.
*   `user_id` (INT) — Уникальный номер клиента.
*   `amount` (INT) — Сумма операции (содержит системные пропуски `NULL` и отрицательные значения возвратов).
*   `date` (TIMESTAMP) — Точная дата и время проведения операции.

---

## Дескриптор авторского ключа (Логика решения):

В ходе технического аудита сырой таблицы `raw_transactions` были выявлены три критические уязвимости данных:
1. Наличие пустых ячеек `NULL`, блокирующих математику сервера;
2. Отрицательные значения, кодирующие операции возвратов;
3. Необходимость жесткой фильтрации временного потока без снижения производительности СУБД.

Для формирования аналитической витрины применена следующая логика:
1. На входе в систему при помощи инструментов `EXTRACT(MONTH)` и `EXTRACT(YEAR)` вырезается целевой отчетный месяц и фиксируется конкретный календарный год. Это обеспечивает элегантную и точную фильтрацию данных в границах периода.
2. Внутри селекта суммы операций `amount` очищаются: при помощи `COALESCE` (в Python Pandas: `.fillna(0)`) убираются пустые ячейки `NULL`, заменяясь нулями, а инструментом `ABS` (в Python: `.abs()`) перехватываются отрицательные минусы возвратов, принудительно трансформируясь в чистое положительное значение оборота. 
3. Чтобы приклеить общую среднюю константу отчетного периода к каждой строчке без склеек и разрушения детального отчета, применяется легкая оконная форсунка `OVER()` без разделений (в Python Pandas эта логика изящно дублируется вектором `.transform('mean')`). Она вычисляет среднее арифметическое по всей майской выборке, округляет его до сотых долей при помощи инструмента смены формата `::NUMERIC(10,2)` и аккуратно клеится в четвертую колонку.
4. На выходе готовый отчет сортируется по убыванию очищенного объема, вынося самые крупные операции наверх реестра.

## СЛОЙ ВИЗУАЛИЗАЦИИ И BI (Проектирование аналитического дашборда)
Очищенная финтех-витрина подключается к BI-платформе (Apache Superset / Yandex DataLens) для оперативного выявления транзакционных аномалий.

### Состав и структура дашборда:
1. **Интерактивный реестр аномалий (Scattered Table):** Детализированная таблица «ID транзакции -> ID пользователя -> Очищенный объем -> Среднее значение за май». Встроенное условное форматирование подсвечивает красным цветом те строки, где очищенный объем превышает среднее значение за период более чем на 200%.
2. **Линейный тренд отклонений (Line Chart):** График, где по оси X отложено время транзакции, по оси Y — объем, а горизонтальная статическая линия отображает расчетную константу среднего значения за май. Позволяет визуально зафиксировать пиковые всплески подозрительной активности в течение месяца.
3. **Модуль быстрой фильтрации по контрагентам:** Панель поиска по конкретному `user_id` для моментального вывода истории его транзакций на экран и сопоставления со средними метриками финансового потока.
