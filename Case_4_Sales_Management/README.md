# Кейс №4: Сепарация весов и фильтрация кубов по VIP-порогу выручки

## Блок: Управление продажами / Коммерческая аналитика
**Заказчик:** Коммерческий директор.

### Бизнес-задача (ТЗ):
> *"Необходимо провести сегментацию клиентской базы и выделить ключевой высокодоходный VIP-сегмент. Требуется рассчитать общие суммы покупок в разрезе контрагентов, учитывая в расчетах исключительно фактически оплаченные заказы. На финальную витрину необходимо вывести только тех клиентов, чей суммарный подтвержденный оборот строго превысил порог в 500 000 рублей."*

---

## Исходная структура СУБД (Таблица `raw_orders`):
*   `order_id` (INT) — Первичный ключ заказа.
*   `client_name` (VARCHAR) — Имя клиента (содержит грязные невидимые пробелы и хаотичный регистр ввода).
*   `status` (VARCHAR) — Текущий статус оплаты заказа (например, 'Paid', 'Canceled').
*   `payment_amount` (INT) — Сумма чека (содержит пропуски `NULL`).

---

## Дескриптор авторского ключа (Логика решения):

В ходе технического аудита сырой таблицы `raw_orders` были выявлены три критические уязвимости данных:
1. Наличие мусорных строк с неотрицательными статусами (отмененные и неоплаченные заказы);
2. Хаотичный ручной ввод имен клиентов с разным регистром и концевыми пробелами, что при обычной группировке искусственно раздробит одного клиента на разные строки;
3. Системные пустоты `NULL` в суммах платежей.

Для формирования корректной витрины применена следующая двухэтажная логика:
1. На самом первом этаже запроса при помощи команды `WHERE status = 'Paid'` срезается неоплаченный мусор строк, отсекая лишние заказы на этапе сырых данных.
2. При помощи `GROUP BY` схлопываем оставшиеся чистые данные по именам клиентов. Чтобы исключить дублирование из-за грязного ввода, применяется инструмент `TRIM(LOWER(client_name))` (в Python Pandas: `.str.strip().str.lower()`), приводящий имена к единому стандарту.
3. Внутри агрегата `SUM(COALESCE(payment_amount, 0))` заполняются пустоты `NULL` нулями, чтобы математика сложения не выдала сбой.
4. И только после, при помощи фильтрационного сита `HAVING` (в Python: последующая фильтрация датафрейма), взвешиваем и оцениваем готовые итоговые веса сумм, оставляя на экране строго VIP-гигантов, чей оборот превысил 500 000 рублей. Финальный отчет сортируется по убыванию витрины.

## СЛОЙ ВИЗУАЛИЗАЦИИ И BI (Проектирование коммерческого дашборда)
Очищенная коммерческая витрина подключается к BI-платформе (Apache Superset / Yandex DataLens) для мониторинга выполнения планов продаж и анализа клиентской базы.

### Состав и структура дашборда:
1. **Рейтинг VIP-клиентов (Leaderboard):** Столбчатая диаграмма (Horizontal Bar Chart), отображающая топ-клиентов по убыванию выручки. Позволяет Коммерческому директору визуально оценить концентрацию капитала и ключевых покупателей.
2. **Индикатор общего VIP-оборота (KPI Card):** Крупный цифровой виджет, суммирующий выручку по всему отфильтрованному сегменту. Показывает, сколько именно денег компании приносят топ-клиенты.
3. **Голографический реестр транзакций (Data Table):** Детализированная интерактивная таблица с возможностью быстрого поиска по имени клиента для оперативного коммерческого аудита конкретного контрагента.
