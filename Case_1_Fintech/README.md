# Кейс №1: Базовая очистка и математическая модернизация данных

## Блок: Финтех
**Заказчик:** Финансовый директор.

### Бизнес-задача (ТЗ):
> *" Данные по операционному доходу за прошлый отчетный период не сходятся с балансом. В выгрузке присутствуют системные сбои (незаполненные ячейки), а также некорректно отображаются возвраты средств, которые занижают показатели общего валового оборота. Нужен чистый реестр транзакций по клиентам с корректным расчетом фактического объема прошедших средств без потери строк."*

---

## Исходная структура СУБД (Таблица `raw_transactions`):
*   `tx_id` (INT) — Первичный ключ транзакции.
*   `user_id` (INT) — Уникальный номер клиента.
*   `amount` (INT) — Сумма операции (содержит пропуски `NULL` и отрицательные значения возвратов).

---

## Дескриптор авторского ключа (Логика решения):

В ходе технического аудита сырой таблицы `raw_transactions` были выявлены две критические уязвимости данных: 
1. Наличие пропусков `NULL`;
2. отрицательные значения, кодирующие операции возврата;

Для формирования корректной витрины валового оборота применена следующая логика:
1. При помощи функции `COALESCE` (в Python Pandas: `.fillna(0)`) убираются пустые ячейки `NULL`, принудительно заменяясь нулями. Это защищает агрегатные функции и математический движок сервера от критических ошибок и падения вычислений в коде.
2. Инструментом `ABS` (в Python: `.abs()`) перехватываются отрицательные маркеры возвратов, которые трансформируются в чистое положительное значение (абсолютную величину). Это позволяет учесть фактический физический объем оборотных средств, прошедших через платформу.

На выходе формируется очищенный реестр с расчетным полем `clean_amount`, готовый к интеграции в BI-слой.




---

## СЛОЙ ВИЗУАЛИЗАЦИИ И BI (Проектирование дашборда Финдиректора)
Для выноса очищенных данных на стол руководства, итоговый структурированный реестр подключается к BI-системе (Apache Superset / Yandex DataLens).

### Конструкция и состав дашборда:
1. **Индикатор «Чистый валовой оборот» (KPI Widget):** Крупный цифровой индикатор, отображающий сумму по всему очищенному полю (`SUM(clean_amount)`). Показывает Финансовому директору реальный объем оборотных средств компании без искажений от системных сбоев и возвратов.
2. **График «Распределение дохода по клиентам» (Bar Chart):** Столбчатая диаграмма, группирующая объемы операций по `user_id`. Позволяет менеджменту мгновенно выявлять наиболее высокодоходных контрагентов.
3. **Реестр сквозной верификации данных (Data Table):** Интерактивная таблица, выводящая связку `tx_id` -> `user_id` -> `clean_amount` с возможностью моментальной фильтрации по конкретному пользователю для проведения оперативного аудита.
